# Правила проекта

## Важные правила

- **Использовать только установленные зависимости** - перед добавлением новых зависимостей проверять `package.json` на наличие уже установленных пакетов. Не устанавливать дублирующие или конфликтующие зависимости.
- **Компоненты shadcn-vue** - разрешается устанавливать и приоритетно использовать все необходимые компоненты из shadcn-vue для построения интерфейса.
- **Не использовать Any** - избегать использования типа `any` в TypeScript/JavaScript коде. Использовать строгую типизацию.
- **Минималистчный код** - писать минималистчный код, но только лучшие практики и быстрые решения.
- **Современный UI/UX** - создавать только современный и красивый пользовательский интерфейс с лучшими практиками UX.
- **TailwindCSS в приоритете** - использовать TailwindCSS как основной инструмент для стилизации. Приоритет отдавать TailwindCSS перед другими CSS-решениями.
- **Старый сайт (prew)** - использовать старый сайт из папки `docs/prew` только как источник данных о товарах и другой контентной информации. Не копировать дизайн или структуру.

## Установленные зависимости и инструменты

- **@types/node** (dev dependency) - типы TypeScript для Node.js
- **shadcn-vue** - библиотека компонентов для Vue.js
  - Стиль: new-york
  - Базовый цвет: neutral
  - CSS переменные: включены
  - Библиотека иконок: lucide-vue-next
  - Алиасы путей настроены (@/components, @/lib/utils, @/components/ui)

## Зависимости shadcn-vue

- class-variance-authority
- clsx
- tailwind-merge
- lucide-vue-next
